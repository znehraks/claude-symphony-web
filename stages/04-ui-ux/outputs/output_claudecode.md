## 3. USER FLOW VALIDATION

### 3.1 Primary Flow (Exploratory Developer)

```
Step 1: LAND
  -> Hero section fills viewport
  -> Particles create premium ambient feel
  -> SplitText animates headline (0.8s)
  -> User reads: "Multi-AI Orchestration Framework"
  -> Time: 0-3s

Step 2: SCAN VALUE PROPOSITION
  -> Subtitle explains what it does (plain language)
  -> Two CTAs visible: copy install command, view on GitHub
  -> User decides: "Interesting, tell me more" (scrolls) or "Let me try" (copies command)
  -> Time: 3-8s

Step 3: SEE QUICK START
  -> Terminal types commands in real-time
  -> Developer recognizes CLI workflow instantly
  -> Builds confidence: "This is a real tool I can use now"
  -> Time: 8-15s

Step 4: UNDERSTAND PIPELINE
  -> 10 stages with clear numbering shows scope
  -> AI model badges show multi-AI approach (differentiator)
  -> Optional: click a stage to see details
  -> Builds understanding: "This is comprehensive and organized"
  -> Time: 15-30s

Step 5: CHECK FEATURES
  -> 6 cards confirm capabilities
  -> Badges highlight what's new/core
  -> Validates: "This solves my problems"
  -> Time: 30-45s

Step 6: VIEW METRICS (SOCIAL PROOF)
  -> Animated numbers create credibility
  -> Concrete data points (stages, AI models, coverage)
  -> Time: 45-55s

Step 7: CONVERT
  -> CTA section offers final nudge
  -> RainbowButton is high-contrast, impossible to miss
  -> User copies install command or clicks GitHub
  -> Time: 55-70s
```

**Validation**: Flow is linear and progressive. Each section answers the next logical question:
- Hero: "What is this?"
- Quick Start: "How do I use it?"
- Pipeline: "What does it do?"
- Features: "Why is it special?"
- Metrics: "Does it work?"
- CTA: "How do I get it?"

### 3.2 Secondary Flow (Developer Shortcut)

```
Step 1: LAND
  -> Read headline, immediately understand it's a CLI tool
  -> Time: 0-2s

Step 2: COPY INSTALL COMMAND
  -> Click copy button in hero
  -> "Copied!" confirmation appears
  -> Open terminal, paste, run
  -> Time: 2-5s total
```

**Validation**: Copy button must be immediately visible and recognizable. The install command text should be readable at a glance (`npx claude-symphony init`). No scroll required. This flow respects the "I already know what I want" developer mindset.

### 3.3 Error and Edge Cases

**JavaScript Disabled**:
- Hero text is server-rendered (thin wrapper pattern), so headline and subtitle are visible
- Terminal shows static command text (no typing animation, but content is readable)
- Pipeline cards render as static grid (no expand behavior, but card content visible)
- Metrics show final numbers (no CountUp, but values present in DOM)
- CTA button is a standard `<a>` tag, works without JS
- **Verdict**: Degraded but fully functional. All content accessible.

**Slow Connection (3G / 500ms+ RTT)**:
- Above-fold content (Hero + Quick Start) loads eagerly (~79KB gzip)
- Below-fold sections show skeleton placeholders while lazy-loading (~50KB gzip)
- Fonts use `display: 'swap'` so system font shows first
- Images: none in the design (all SVG icons inline or Lucide components)
- **Verdict**: Usable within 3-5s even on slow connections. Skeletons prevent layout shift.

**Mobile User (touch device)**:
- All tap targets minimum 44x44px
- Pipeline cards: full-width tap targets, detail panel slides down inline
- Copy button: large enough to tap, haptic feedback via device default
- No hover-dependent information (all hover effects are enhancement only)
- Terminal: scrollable if content overflows horizontally
- **Verdict**: Fully functional. See Section 6 for responsive detail.

### 6.2 Touch Target Compliance

| Element | Desktop Size | Mobile Size | Meets 44x44px? |
|---------|-------------|-------------|-----------------|
| Copy command button | ~200x40px | Full width, 48px height | Yes |
| GitHub link button | ~140x40px | Full width, 48px height | Yes |
| Pipeline stage card | ~200x180px | Full width, ~120px height | Yes |
| RainbowButton CTA | ~240x48px | Full width, 52px height | Yes |
| Secondary CTA link | Text link, ~16px | Add `py-3` for 48px touch target | Needs padding |

**Issue Found**: Secondary text links in the CTA section need explicit vertical padding on mobile to meet 44x44px minimum. Recommendation: apply `py-3` on mobile breakpoint.

### 6.3 Content Prioritization by Breakpoint

**Mobile (<640px)**:
- Hero: headline text-4xl (reduced from text-6xl), subtitle text-base, CTAs stacked vertically full-width
- Quick Start: terminal full-width with horizontal scroll for long commands
- Pipeline: single column stack, no connectors, cards show condensed info (name + AI badge only), detail panel inline below card
- Features: single column, cards full-width
- Metrics: 2x2 grid (compact)
- CTA: full-width, stacked elements
- Section padding: `py-16` (reduced from `py-24`)

**Tablet (640-1023px)**:
- Hero: headline text-5xl, CTAs side-by-side
- Quick Start: terminal with increased horizontal padding
- Pipeline: 2-column grid (5 rows), no connectors
- Features: 2-column grid (3 rows)
- Metrics: 4-column row (tight)
- CTA: increased max-width
- Section padding: `py-20`

**Desktop (>=1024px)**:
- Hero: headline text-6xl, max-w-4xl centered
- Quick Start: terminal max-w-[720px] centered
- Pipeline: 5-column x 2-row grid with CSS connectors between cards
- Features: 3-column grid (2 rows)
- Metrics: 4-column row with large numbers
- CTA: max-w-[640px] centered
- Section padding: `py-24`

### 6.4 Gesture Support

| Gesture | Platform | Behavior |
|---------|----------|----------|
| Scroll (vertical) | All | Natural page scrolling, triggers IntersectionObserver animations |
| Tap | Touch devices | Activates buttons, toggles pipeline cards |

### 6.5 Navigation Pattern

This is a single-page landing site with no traditional navigation bar. Navigation is purely scroll-based.

- **Desktop**: Smooth scroll through sections. Optional: subtle scroll indicator in hero ("Scroll to explore")
- **Mobile**: Standard touch scroll. No hamburger menu needed (single page)
- **Keyboard**: Tab through interactive elements follows DOM/visual order. Skip-to-content link at top.

### 6.6 Mobile-Specific Adjustments

| Adjustment | Reason |
|------------|--------|
| Particles reduced from 40 to 20 | Performance on lower-end devices |
| Mouse tracking disabled on particles | Touch devices have no persistent pointer |
| MagicCard cursor gradient disabled | No hover on touch; show static subtle gradient instead |
| Pipeline connectors hidden | Too small and complex on single-column layout |
| Terminal horizontal scroll enabled | Long commands may exceed viewport width |
| Section padding reduced by ~33% | Tighter vertical rhythm on small screens |
| Font sizes scaled down one step | Maintain readability without excessive scrolling |
