## 5. User Flow

### Primary Journey

1. **Awareness** (0-3s): User lands, sees "Orchestrate your entire development lifecycle". Understands it's a CLI tool via terminal visual and particles background.
2. **Quick Validation** (3-8s): Scrolls to Quick Start. Sees terminal typing animation. Recognizes familiar CLI interface. Trust established.
3. **Investigation** (8-20s): Scrolls to Pipeline. Hovers over stage cards. Clicks "06 Implementation" to see it generates code. Clicks "08 QA" to see it runs tests. Understands the full workflow.
4. **Value Assessment** (20-30s): Scans Features grid. Notes "Multi-AI Orchestration", "Sub-Agent System", "Sprint Mode". Cross-references with pipeline understanding.
5. **Social Proof** (30-35s): Sees metrics: "10 Pipeline Stages", "109+ Tests", "<5ms Speed", "0% Context Usage". Quantitative validation.
6. **Conversion** (35-45s): Reaches CTA. Copies `npx claude-symphony my-project` or clicks GitHub link.

### Secondary Flow (Developer Shortcut)

1. Land -> See install command in Hero -> Copy immediately -> Done (under 5 seconds)

### Edge Cases

- **JS Disabled**: Static HTML renders from SSR. Hero text visible. Copy command shows as text. No animations.
- **Slow Connection**: Skeleton loaders for lazy sections. Above-fold loads first (~79KB).
- **Mobile**: Single-column layout, simplified pipeline (no connectors), larger touch targets.
