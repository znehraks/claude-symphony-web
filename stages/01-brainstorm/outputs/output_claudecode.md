### 1.3 SSR/Client Boundary

Every section needs `"use client"` due to animations. Recommended: thin Server Component wrappers for static text, Client sub-components for animated elements. This reduces hydration cost.

## 2. Performance Risk Analysis

### Risk Matrix

| Animation | Risk | Mitigation |
|-----------|------|------------|
| Particles (Canvas, 40) | MEDIUM | requestAnimationFrame + IntersectionObserver pause |
| Magic Card (mouse tracking) | MEDIUM | Throttle to 60fps, `will-change: transform` |
| Flickering Grid | HIGH | Use Canvas with staggered updates, 5% cells/frame |
| SplitText | MEDIUM | Limit to hero headline only |
| AnimatePresence | MEDIUM | Keep exit animations <300ms |
| Border Beam, Retro Grid | LOW | Pure CSS, GPU-composited |

### IntersectionObserver Strategy

| Section | Threshold | Enter | Exit |
|---------|-----------|-------|------|
| Hero Particles | 0 | Start loop | Pause loop |
| QuickStart | 0.2 | Start typing | Keep state |
| Pipeline | 0.1 | Stagger cards | Keep visible |
| Features | 0.1 | Stagger cards | Keep visible |
| Metrics | 0.3 | Start CountUp | Keep numbers |
| CTA | 0.2 | Animate gradient | Keep animating |

**Critical**: Canvas loops (Particles, Flickering Grid) MUST pause when off-screen.

## 4. Technical Recommendations

### Must-Have Animations
- Particles background (visual identity)
- SplitText headline (first impression)
- Pipeline Visualizer cards (core value prop)
- CountUp numbers (social proof)
- CTA buttons (conversion)
- Typing animation (developer credibility)

### Nice-to-Have (cut if tight)
- Animated Beam connections → fallback: CSS dashed lines
- Flickering Grid → fallback: CSS gradient background
- Border Beam on Feature cards → fallback: subtle border
- Aurora Text → fallback: static gradient text
- Orbiting Circles → not needed for current plan

### Reduced-Motion Strategy
- Framer Motion: automatic respect via `motion` components
- Canvas (Particles, Grid): render single static frame
- Text effects: show instantly without animation
- CountUp: show final number immediately

### Mobile Concerns
- Particles: reduce 40 → 20 on mobile
- Pipeline: single column, no beams
- Magic Card: static gradient on touch (no mouse tracking)
- Canvas DPI: cap at 2x `devicePixelRatio`

## 5. Alternative Approaches

### Simplifications That Preserve 90% Visual Impact

| Planned | Simpler Alternative | Effort Saved |
|---------|---------------------|--------------|
| Animated Beam (SVG) | CSS dashed border with `animation: dash` | 4-6 hours |
| Flickering Grid (Canvas) | CSS radial-gradient with opacity spots | 3-4 hours |
| SplitText (per-character) | Word-level stagger with Framer Motion | 1 hour |
| Number Ticker (digit-by-digit) | Simple CountUp with easing | 1 hour |

### Over-Engineering Warnings
1. **Pipeline Animated Beams**: Highest complexity-to-impact ratio. CSS connectors work equally well.
2. **Flickering Grid**: Full Canvas for a background effect is disproportionate. CSS alternative recommended.
3. **Multiple text animation types**: Standardize on 2 max (SplitText for hero, CSS gradient for titles).
